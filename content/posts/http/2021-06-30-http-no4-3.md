# HTTP 메시지
- - - - 


## HTTP 메시지 구조


**공식 스펙**
```
start-line
*( header-field CRLF )
CRLF
[ message-body ]
```


```
start-line 시작라인
header 헤더
empty line 공백라인(CRLF)
message body
```

HTTP 요청 메시지에는 body가 있을수도 있고 없을 수도 있음



### 요청 메시지의 start-line

```
GET /search?q=hello&hi=ko HTTP/1.1
Host: www.google.com
```

    - GET          : HTTP 메서드 
    - 요청 대상      : /search?q=hello&hi=ko    (절대경로) 
    - HTTP Version : HTTP/1.1




### 응답 메시지의 start-line

```
HTTP/1.1 200 OK
Content-Type: text/html;charset=UTF-8
Content-Length: 3423

<html>
    <body>....</body>
</html>
```

    - HTTP/1.1 : HTTP 버전
    - 200 : HTTP 상태코드
    - OK : 사람이 이해 할 수 있는 짧은 상태 코드 설명 글



### HTTP 헤더

헤더에는 바디를 제외한 필요한 모든 메타데이터가 들어가 있음

field-name 과 value로 이루어져 있으며 field-name은 대소문자를 가리지 않음.

예를 들어 요청메시지의 헤더에 Host는 field-name이고, www.google.com은 value.



### HTTP 메시지 바디

byte로 표현 할 수 있는 모든 데이터 전송 가능

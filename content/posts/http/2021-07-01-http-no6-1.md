# HTTP 메서드 활용_클라이언트에서 서버로 데이터 전송
- - - - 


### 클라이언트에서 서버로 데이터 전송

##### 데이터 전달 방식 2가지

    1. 쿼리 파라미터를 통한 데이터 전송
    
        - GET
    
    
    2. 메시지 바디를 통한 데이터 전송

        - POST, PUT, PATCH


##### 클라이언트에서 서버로 데이터를 전송하는 4가지 상황

    1. 정적 데이터 

        이미지, 정적텍스트 문서

        정적데이터는 쿼리파라미터 없이 리소스 경로로 단순하게 조회 가능


    2. 동적 데이터 
    
        검색, 게시판 목록(검색어)

        
    3. HTML FORM을 통한 데이터 전송 
        
        - 회원가입, 상품주문, 데이터 변경
        
        - Form에는 GET, POST만 사용이 가능. 데이터 조회가 아니라면 GET을 사용하지 않도록 함


        - <form action="/save" method="get">      ====> 조회

        - <form action="/save" method="post">     ====> 저장

        - <form action="/save" method="post" enctype="multipart/form-data">     ====> 파일전송
        


        - Content-Type: application/x-www-form-urlencoded

            쿼리 파라미터와 유사한 형식으로 데이터를 만들고 HTTP 바디에 넣고 인코딩 되어서 넘어간다



    4. HTTP API를 통한 데이터 전송

        - 회원가입, 상품주문, 데이터 변경
        
        - 서버 to 서버
            
            백엔드 시스템 통신

        - 앱 클라이언트
            
            아이폰, 안드로이드

        - 웹 클라이언트(ajax)
            
            HTML에서 Form전송 대신 자바 스크립트를 통한 통신에 사용(AJAX). 
            
            React, VueJs같은 웹크라이언트와 API 통신

            Content-Type:application/json을 주로 사용
